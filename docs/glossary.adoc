Naming authority for the Urd engine and component layer.

---

== Domain

[glossary]
[[domain-component]]Component::
A self-contained facade for a specific domain model.
Takes a <<domain-parameter,parameter>> set at initialisation.
Hosts a <<domain-dynacore,dynamical core>> that advances <<domain-state,state>> over time.
Exposes that state as read-only.
Responds to <<domain-forcing,forcings>>.

[[domain-coupling-frequency]]Coupling Frequency::
Earth-system modelling term for how often a model component exchanges state with its neighbours.
Used in Urd as the scientific basis for <<engine-cadence,cadence>>.

[[domain-dynacore]]Dynamical Core (DynaCore)::
The computational core of a <<domain-component,component>>.
Owns state, advances it over time, and receives <<domain-forcing,forcings>> from its component.
Callers interact with the component, not the dynamical core directly.

[[domain-forcing]]Forcing::
An external perturbation introduced into a running simulation.
The standard Earth-system term for a stressor applied from outside a component's own dynamics — solar output variations, volcanic aerosols, and designer-injected events are all forcings.

[[domain-parameter]]Parameter::
A simulation input.
Typed, validated value provided to a component at initialisation time.
Parameter is frozen once the simulation starts; runtime changes enter through <<domain-forcing,forcings>>.

[[domain-state]]State::
A simulation output.
An observable value produced by a component as it advances through time.
State is read-only to consumers; only the owning component may update it.

== Engine

[glossary]
[[engine-cadence]]Cadence::
Project term for <<domain-coupling-frequency,coupling frequency>>.
How often a <<domain-component,component>>'s <<domain-dynacore,DynaCore>> advances state.
Cadence is intrinsic to the component's physical domain — atmospheric circulation ticks faster than tectonic deformation.
Components with different cadences naturally stagger: a slower component consumes the latest state from a faster one on its next tick, without explicit coordination.

== Topology

[glossary]
[[topology-grid]]Grid::
The spherical partition abstraction.
Provides deterministic, side-effect-free operations over a hierarchy of cells covering the sphere.
All coordinates are latitude/longitude in degrees; cell identifiers are opaque `CellId` values.
+
_Source type:_ `IGrid`.

[[topology-resolution]]Resolution::
A discrete level in the cell hierarchy. `R0` is the coarsest level; higher values are finer.
The valid range and subdivision factor are backend-specific.
Components operate over a resolution range appropriate to their physical scale — macro processes use coarser resolutions; local detail uses finer ones.
+
_Source type:_ `Resolution`.

[[topology-cell]]Cell::
An opaque `CellId` identifier for one node in the grid hierarchy.
A cell has a resolution, a centre coordinate, a boundary polygon, a parent at any coarser resolution, and children at any finer resolution.
Validity is checked with `IGrid.IsValid`.

[[topology-latlng]]LatLng::
A latitude/longitude coordinate in degrees, used as the common spatial currency of the topology layer.
+
_Source type:_ `LatLng`.
