= Topology Reference
:toc: left
:toclevels: 2

== Purpose

Defines the spherical partition used by all simulation processes.
The backend is opaque; all access is through `IGrid`.

== Files

[cols="1,2"]
|===
|File |Role

|`CellId.cs`
|Opaque cell identifier. Wraps the backend representation.

|`IGrid.cs`
|Operation contract. Only interface simulation code may depend on.

|`LatLng.cs`
|Value object. Latitude/longitude in degrees as two `double` primitives.

|`Resolution.cs`
|Resolution value type. R0 is coarsest. Valid range is backend-specific.

|===

== Contract (`IGrid`)

All operations are deterministic and side effect free.
Cell identifiers are opaque `CellId` values.
Coordinates are `LatLng` (degrees).
Invalid input must not crash callers; implementations return safe sentinel values and emit diagnostics.

[cols="1,2"]
|===
|Operation |Description

2+h|Partition access

|`CellAt(lat, lng, resolution)`
|Resolve cell from raw degree values and resolution.

|`CellAt(latLng, resolution)`
|Overload accepting a `LatLng` value directly.

|`CellsAtResolution(resolution)`
|All cells at a given resolution. Expensive at high resolutions — see `Resolution.cs`.

|`RootCells()`
|Complete top-level partition (R0).

2+h|Cell introspection

|`BoundaryOf(cell)`
|Boundary polygon vertices in traversal order.

|`CenterOf(cell)`
|Centre coordinate of a cell.

|`IsValid(cell)`
|Whether a cell identifier is valid.

|`ResolutionOf(cell)`
|Resolution of a given cell.

2+h|Hierarchy traversal

|`ChildrenOf(cell, resolution)`
|All child cells at a finer resolution.

|`InferResolution(cells, fallback)`
|Resolution of the first key in a dictionary; fallback if empty.

|`ParentOf(cell, resolution)`
|Ancestor cell at a coarser resolution.

2+h|Neighbourhood traversal

|`Disk(cell, k)`
|All cells within ring radius k, including centre.

|`GridRing(center, k)`
|Cells at exactly ring distance k from centre. k=0 returns centre only.
|===

== Resolution model

* R0 is the coarsest level, covering the full sphere.
* Higher values are finer. The valid range and subdivision factor are backend-specific.
* Full enumeration via `CellsAtResolution` grows rapidly. Prefer localised queries (`Disk`, `GridRing`, `ChildrenOf`) at fine resolutions.
